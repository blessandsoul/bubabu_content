# Bubabu Smart Gifts, production-магазин (переделан 1:1 под референс)

## Что это
Multi-file production-макет главной, выровненный по референсу `Downloads/ChatGPT Image 17 июн 19_40_05.png` (тёплый премиум-ритейл «подарки для всех возрастов»). Русский, реальный копирайт, сгенерённая фотография. Демо-концепт для показа.

## Как открыть
```
cd agents/bubabu/draft/20260617_wed_bubabu_store_pro
python -m http.server 8080
```
http://127.0.0.1:8080 . Нужен интернет для шрифта Manrope + иконок Phosphor (CDN); фото локальные.

## Переделка под референс (после «найди 20 различий»)
Было: длинный лендинг с лишними секциями. Стало: лаконичный каталог как референс. **Закрыто 20 различий**, главное:
- HERO теперь ОДНА фото-карточка: тёплая сцена (коробки/наушники/ёлка-боке) + сова Bubabu по центру-справа + текст слева («Подарки для всех возрастов» / «Умные. Оригинальные. Для каждого.» + чипы + 2 кнопки + play-строка). Раньше был split.
- «Подарки для X» = ряд из 7 МЕЛКИХ карточек (было 6 больших портретов).
- Хиты = 6 в ряд + рейтинг с числом отзывов + скидки (было 5, только звёзды).
- «Почему» = инлайн иконки (было карточки).
- По случаю = мелкие чипы; Nav = 10 пунктов; лого = «Bubabu Smart Gifts».
- v3: БЕЛЫЙ фон + тёплые акценты (было всё бежевое). Хедер: поиск на всю ширину (был прижат влево), меню переосмыслено (оранжевая бренд-пилюля «Bubabu AI друг» + «Скидки» справа, заполняет ширину). Doc-блоки ВЕРНУЛ компактно на белом: Gift Finder, Популярные категории, Подарочные наборы, Отзывы.
- УБРАНЫ только: promo-marquee, free-ship progress-bar, отдельный флагман-бэнд (сова в hero+хитах), видео-бэнд, new-arrivals rail (в nav). Остальные doc-блоки на месте.

## Обновление: настоящий mobile-first (нативный мобильный слой)
Претензия: сайт был спроектирован под ПК и просто ужимался на мобиле (мелкие 2-кол сетки). Сделан нативный мобильный слой (<900px), десктоп (≥900) НЕ тронут (сбрасывается полностью):
- **Нижний таб-бар** (Главная / Каталог / Подбор / Корзина•бейдж / Профиль) fixed внизу, иконки+подписи. Главный нативный сигнал. Скрыт на ≥900. `body padding-bottom:64px` на мобиле.
- **Товарные ряды = горизонтальные свайп-карусели** на телефоне (`.giftrow/.pgrid/.cats/.ages/.sets/.revs` -> flex + scroll-snap, карточки фикс-ширины с «подглядыванием» следующей). На ≥760 возвращаются в сетки, на ≥900/1200 десктоп-колонки.
- **Чистая мобильная шапка**: десктопный topbar (4 траст-пункта) скрыт <900, компактные паддинги/лого. На ≥900 возвращён.
- **Меню = оверлей-панель** из шапки (крупные тап-строки, скролл), а не тесный инлайн-дроп; закрывается по тапу на пункт.
- **PDP мобайл**: галерея = скруглённый квадрат + лента тамбов, свайп по фото листает кадры; sticky-buy `z-70` поверх таб-бара.
- Реализация: мобильная база в style.css + сбросы в responsive.css @760/@900; JS (свайп фото, закрытие меню) под guard'ами.
- **QA:** Playwright 360/390 (мобайл) + 1440 (десктоп) обеих страниц: таб-бар flex<900 / none≥900, ряды свайпаются на телефоне и сетки на десктопе, topbar скрыт/показан, page h-scroll = 0 везде (карусели и таблица сравнения скроллятся ВНУТРИ себя), десктоп пиксельно как был (giftrow 7 / pgrid 6 / PDP 2 кол). em-dash 0.
