01 · РОЛЬ

Bubabu = маскот магазина

Bubabu появляется в каждой рекламе. В кадре с товаром — держит / рассматривает / реагирует. Не один из 37 SKU — он надо всеми.

02 · ГОЛОС

Голос не меняется

03 · ТОВАРЫ

SKU без брендинга

Магниты = просто магниты в руках у Bubabu. Никаких имён, никаких character.md для не-Bubabu товаров. Photo + Bubabu + 1 строка пользы + цена.

04 · МИР

Один мир, один нарратив

У Bubabu в облачном дереве есть полка с подарками. Он берёт один, спускается к ребёнку, отдаёт. 37 товаров каталога = его полка.

05 · ТЕМП

Чёт-нечёт график

Чётный день = новый мульт. Нечётный = реклама из вчерашнего мульта. 37 SKU × 2 дня = ~10 недель полный охват каталога.

Pair-folder

Workflow одной пары

Что НЕ меняется + Известные риски

Пивот — про коммерцию. Бренд Bubabu (как герой) остаётся целостным. Риски осознаны.

Что осталось ровно как было

  • Внешность Bubabu (белая пушистая сова, циан-очки, чёрный клюв закрыт навсегда)
  • Reference jpeg-файлы (1.jpeg / 2.jpeg / 3.jpeg) — обязательны на каждом рендере
  • Голос: Bubabu молчит, narrator Charon Storytime
  • Веками отлаженные production-локи: статичная камера, никаких squash/stretch, Pixar 3D не photoreal
  • Языковые правила: EN+KA only, никогда Russian/Cyrillic в кадре
  • Draft-first до «выложил»
  • Произношение .ge = «ჯი» в TTS

Известные риски

  • 37 мультов = большая производственная нагрузка. Срыв темпа → пара без cartoon
  • Wonder Tools реконтекст может выглядеть натянуто (Wisdom Box vs «лептоп»). Тест на первой паре
  • Дети в мультах разные каждую серию (нет face-lock между ep) — может ощущаться как «у Bubabu нет постоянного друга»
  • Premortem skipped — если через 6 мес видим что-то ломается, нужна reflection-engine ретроспектива
  • Flagged SKU (дрон model mismatch, магнитные шарики safety) НЕ продвигаются до снятия флага
  • Public announce пока НЕ делаем — пилот retail-only
